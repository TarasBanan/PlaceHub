# ТЗ: сайт-платформа для бронирования городских пространств «PlaceHub»

## 1. Название проекта

**PlaceHub** — онлайн-платформа для поиска, сравнения и бронирования городских пространств: коворкингов, переговорных комнат, фотостудий, залов для мероприятий, лекционных аудиторий, pop-up площадок и небольших event-пространств.

Проект должен выглядеть как полноценный коммерческий продукт для реального сервиса бронирования и подходить для портфолио frontend/fullstack-разработчика.

---

## 2. Цель проекта

Разработать многостраничный сайт для сервиса, который помогает пользователям быстро находить и бронировать подходящие пространства под рабочие встречи, съемки, лекции, мастер-классы, презентации, корпоративы и частные мероприятия.

Сайт должен решать две задачи:

1. Для клиента — быстро найти площадку по параметрам, посмотреть фото, условия, цену и отправить заявку на бронирование.
2. Для владельца площадки — показать преимущества сервиса и предложить добавить свою площадку в каталог.

---

## 3. Основная идея сайта

Сайт представляет PlaceHub как маркетплейс городских пространств. Пользователь должен за 1–2 минуты понять:

- какие типы площадок доступны;
- сколько стоит аренда;
- где находится площадка;
- какие есть удобства;
- свободна ли площадка на нужную дату;
- как оформить заявку;
- почему безопаснее бронировать через сервис, а не напрямую.

Главная ценность проекта — удобный каталог с фильтрами, карточками площадок, страницами объектов, калькулятором стоимости и формой бронирования.

2. Укажите дату и количество гостей

## 6.4. Избранное

Пользователь может добавлять площадки в избранное.

Минимальная реализация:

- хранить избранное в localStorage;
- менять состояние кнопки;
- показывать отдельный список избранных площадок.

---

## 6.5. Калькулятор стоимости

Калькулятор должен менять итоговую примерную цену в зависимости от выбранных параметров.

Формула:

`итог = цена_за_час * количество_часов + дополнительные_услуги`

Если выбран кейтеринг:

`кейтеринг = цена_за_гостя * количество_гостей`

Итоговая стоимость должна отображаться сразу после изменения параметров.
